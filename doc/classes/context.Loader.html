context.Loader Class

Defined in: src\Loader.js:8
Module: context

Manages asynchronous asset loading with progress feedback.

To load assets using the Loader you must first queue loading tasks using the provided queueXxx methods and then start the loading process using the loadAssets method.

The Loader is able to load textures, shader pairs and generic data such as XML files and JSON objects.

Textures are loaded using the AsyncTexture class. Shader pairs are loaded using the AjaxProgram class and thus they are automatically compiled and linked, throwing an exception if compilation or linking fails.

XML files are loaded via Ajax and returned as DOM Document objects. JSON files are returned as natural JavaScript objects.

loadAssets

(
  • callback
  • [progress]
)
chainable

Defined in src\Loader.js:363

Executes the load tasks queued so far and discards the queue. When the loading process finishes, a new Assets object is created and can be used to retrieve the loaded assets.

The same Loader object can be used to load multiple independent asset sets: assets belonging to different sets must be queued for different loadAssets calls. Each Assets object does not affect the others.

Parameters:

  • callback Function

    A user-defined callback function invoked by the Loader as soon as all the loading tasks have been accomplished.

    • assets context.Loader.Assets

      A new Assets object that can be used to retrieve and manage the loaded assets.

  • [progress] Function optional

    A user-defined callback function invoked several times during the loading process. It can be used to provide progress feedback to the user.

    • progress Number

      The current progress percentage.

Queues an asynchronous task that loads, compiles and links the specified shader pair, and returns it as a Program object.

Internally the queueProgram method uses the AjaxProgram class.

The loaded program can then be retrieved using the context.Loader/getProgram method.

Parameters:

  • id String

    The URL of the GLSL shaders to load, excluding the filename extension which is automatically appended (.frag for fragment shaders and .vert for vertex shaders).

  • attributes String[]

    An array of attribute array variable names to associate to array indices. See the AjaxProgam class for more information.

queuePrograms

(
  • map
)
chainable

Defined in src\Loader.js:215

Queues asynchronous tasks that load, compile and link zero or more shader pairs, and return them as Program objects.

Internally the queuePrograms method uses the AjaxProgram class to load each shader pair.

The loaded programs can then be retrieved using the context.Loader/getProgram method.

Parameters:

  • map Object

    A dictionary object that associates shader pair URLs to attribute arrays. For example:

    {
    "glsl/box": ["in_Vertex", "in_TexCoord"]
    "glsl/sphere": ["in_Vertex", "in_Normal", "in_TexCoord"]
}

    In this example we assumed there is a glsl directory containing two shaders pairs: box.vert, box.frag, sphere.vert and sphere.frag. Besides, the box shaders use two attribute arrays which are in_Vertex and in_TexCoord, while the sphere shaders use three attribute arrays which are in_Vertex, in_Normal and in_TexCoord.
